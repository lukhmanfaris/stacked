Code Review Findings

Critical

Export endpoint — no pagination / memory risk

Export fetches ALL bookmarks in one query. Pro users with 10K bookmarks could hit memory limits or timeout on serverless.

// api/export/route.ts const { data } = await supabase .from('bookmarks') .select('*') // no .range() — unbounded

src/app/api/export/route.ts

Critical

Search endpoint — no RLS ownership filter

Full-text search on search_vec must explicitly filter by user_id, or rely solely on RLS. Verify RLS policy covers textSearch path — supabase's textSearch() can bypass policies if misconfigured.

// api/search/route.ts — confirm RLS active // or add explicit: .eq('user_id', user.id)

src/app/api/search/route.ts

Warning

Bulk action — 200 ID limit not server-validated

Client sends up to 200 IDs. Zod schema should validate ids.max(200). Without it, attacker can send 10K IDs in one request.

// validators.ts — add: ids: z.array(z.string().uuid()).min(1).max(200)

src/app/api/bookmarks/bulk/route.ts

Warning

SSR localStorage access — sidebar + view mode

DashboardContext reads localStorage synchronously on init. Will throw in SSR/Edge environments if not guarded.

// dashboard-context.tsx // Needs: typeof window !== 'undefined' guard // or initialise with null → read in useEffect

src/contexts/dashboard-context.tsx

Warning

Metadata edge function — no timeout fallback UI

8s timeout defined in constants. If edge function fails/times out, bookmark is still created with empty metadata. User sees no indication it failed silently.

// METADATA_FETCH_TIMEOUT = 8000 // Bookmark created regardless — silent degradation

src/lib/constants.ts · src/app/api/bookmarks/route.ts

Warning

Shared link slug — 5 retries not enough at scale

Auto slug generation tries up to 5 suffixed variants. With many public users, collisions become likely. Should fallback to UUID segment after exhaustion.

// shared-links/route.ts // After 5 attempts → should use nanoid fallback // Currently throws ConflictError

src/app/api/shared-links/route.ts

Warning

useBookmarks — filter state reset on remount

Filter state lives in hook memory. On component remount (route change → back), filters reset to DEFAULT_FILTERS even though URL params may have the state. URL params should be source of truth.

// use-bookmarks.ts const [filters, setFilters] = useState(DEFAULT_FILTERS) // Should initialise from URL params

src/hooks/use-bookmarks.ts

Warning

Hard delete — domain orphan cleanup is best-effort

maybe_orphan_domain RPC called after delete. If it fails, orphan domains accumulate in storage. No retry or error surfacing.

// api/bookmarks/[id]/route.ts DELETE await supabase.rpc('maybe_orphan_domain', {...}) // failure here = silent orphan leak

src/app/api/bookmarks/[id]/route.ts

Good

Consistent error hierarchy

AppError → ValidationError, NotFoundError, TierLimitError etc. All map to specific HTTP status codes. handleApiError() centralises all error handling — no scattered try/catch patterns.

src/lib/errors.ts · src/lib/api-response.ts

Good

Zod validation at every API boundary

All POST/PATCH routes validate with Zod before any DB call. Type safety flows from schema to TypeScript types via z.infer. Field-level errors returned in ValidationError response.

src/lib/validators.ts

Good

Tier limits enforced before mutations

checkTierLimit() called before INSERT on bookmarks, categories, shared_links. Returns resource + limit + current count in error — good for client-side messaging.

src/lib/tier.ts

Good

Soft delete with trash state

Bookmarks are soft-deleted to trash (is_trashed), then optionally hard-deleted. Prevents accidental data loss. Trash view lets users recover before permanent deletion.

src/app/api/bookmarks/[id]/route.ts

Good

Duplicate URL detection with warning (non-blocking)

On bookmark create, checks for existing URL. Returns duplicate_warning flag + existing_id without blocking creation. User can choose to proceed or navigate to existing.

src/app/api/bookmarks/route.ts

Good

React Query cache invalidation strategy

Bookmark mutations invalidate both ['bookmarks'] and ['bookmark-counts'] in one shot. No stale count bugs. Category mutations also invalidate counts.

src/hooks/use-bookmarks.ts

Good

Web Locks API in UserContext

Uses navigator.locks.request to prevent concurrent profile fetches during rapid auth state changes. Prevents race conditions with a 'cancelled' flag pattern.

src/contexts/user-context.tsx

Good

Pinned bookmarks always sort first

Both GET /api/bookmarks and GET /api/search apply .order('is_pinned', { ascending: false }) before secondary sort. Consistent across all views.

src/app/api/bookmarks/route.ts · src/app/api/search/route.ts

Info

No rate limiting on API routes

RateLimitError class exists in errors.ts but no middleware applies it. Consider adding rate limiting on /api/metadata and /api/import to prevent abuse.

src/lib/errors.ts

Info

search_vec column — no migration visible

Search uses PostgreSQL textSearch on search_vec. This column requires a generated tsvector column + GIN index in migrations. Verify it's created in DB schema — not visible in migrations 006-008.

supabase/migrations/ · src/app/api/search/route.ts
